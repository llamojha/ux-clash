# Session — 2026-03-23

## Summary
Implemented Phase 3 (Shareability + Public Pages) of UX Clash. Added a public submission detail page, share buttons, dynamic OG images for social media previews, clickable submission cards, and a submission confirmation dialog. Conducted a code review and applied fixes for security, reliability, and accessibility issues.

## Key Decisions
- Public submission page at `/submission/[id]` with split layout: interactive full-size iframe (left) + metadata sidebar (right)
- Share button uses `navigator.share()` on mobile, falls back to clipboard copy + "Enlace copiado" toast on desktop
- Dynamic OG images via Next.js `ImageResponse` (Satori) — text-based cards (no screenshots), falls back to static `/og.png` on error
- Submission cards use overlay link pattern (absolute `Link` + `z-10` on LikeButton) to stay as server components
- `buildSrcdoc` extracted to `src/lib/srcdoc.ts` as shared utility (used by `SubmissionPreview` and submission detail page; `PreviewPanel` still has its own copy)
- White background (`background:#fff`) baked into `<body>` in srcdoc HTML so previews match editor
- Confirmation dialog (shadcn AlertDialog) added before submission: "Una vez enviada, no podrás editarla"
- No edit capability for submissions — intentional design decision
