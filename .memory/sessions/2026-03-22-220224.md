# Session: Phase 0 — Foundation + Design System

**Date:** 2026-03-22 22:02 CET

## Summary
Planned and implemented Phase 0 of UX Clash — a competitive UX/UI arena for the CubePath 2026 hackathon (deadline March 31). Went from zero code to a fully scaffolded Next.js 16 project with Supabase auth, DB schema, custom design system, layout components, and skeleton pages. Also did a code review and fixed a security issue.

## Key Decisions
- **Next.js 16.2** (not 15 as originally in roadmap) — React 19, faster dev/rendering
- **Bun** as package manager and runtime (installed via npm, v1.3.11)
- **Supabase Cloud** for auth + DB instead of self-hosted PostgreSQL + Auth.js — simpler, faster, free tier sufficient
- **No ORM** — Supabase JS client only with generated TypeScript types
- **"Modern Editorial" design** — clean, minimal, dark-mode-first, Linear/Vercel-inspired (not "gritty dark arena")
- **Geist Sans + Geist Mono** — dropped JetBrains Mono (one fewer font, Monaco editor uses its own font anyway)
- **shadcn/ui base-nova style** — uses @base-ui/react (no `asChild` prop, uses `render` prop instead)
- **Dokploy + Railpack** for CubePath deployment (not Coolify)
- **One CubePath VPS** (Dokploy for app) + **Supabase Cloud** (auth + DB) — recommended against self-hosting Supabase
