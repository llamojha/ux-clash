## Summary
Planned and implemented Phase 1 of UX Clash — the editor-to-submit core loop. Confirmed that 3 seed challenges from Phase 0 are sufficient for testing — the full 10 challenges come in Phase 5.

## Key Decisions
- **@monaco-editor/react@next** (v4.8.0-rc.3) — React 19 compatible, multi-model via `path` prop for HTML/CSS tabs
- **monaco-editor** v0.55.1 as peer dep for types
- **Tailwind v4 Play CDN** (`@tailwindcss/browser@4`) loaded inside sandboxed iframe via `srcdoc`
- **iframe sandbox="allow-scripts"** without `allow-same-origin` — needed for Tailwind CDN, isolates from parent
- **HTML sanitization** — shared `src/lib/sanitize.ts` strips `<script>` tags and `on*` attributes, used client + server
- **One submission per challenge** — unique constraint on `(user_id, challenge_id)`, enforced at DB + API level
- **Standard shell kept** — editor page uses SiteHeader/SiteFooter from root layout (user chose option c)
- **Stay on editor after submit** — success toast via Sonner, no redirect (user chose option a)
- **One submission only, final** — no edits or resubmits (user chose option a)
- **Sonner simplified** — removed `next-themes` dependency, hardcoded dark theme
- **Cmd/Ctrl+S** shortcut triggers submit
- **Login redirect** — OAuth redirects back to `/editor/[slug]` via `?next=` param
- **No additional seed challenges needed** — 3 from Phase 0 cover all test cases (template/no-template, mobile/desktop, daily/weekly)
